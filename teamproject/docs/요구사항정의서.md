# 요구사항정의서

## 1. 프로젝트 개요

### 1.1 프로젝트명
**튜터 매칭 플랫폼 (Tutor Matching Platform)**

### 1.2 프로젝트 목적
학생과 튜터를 연결하는 온라인 1:1 과외 매칭 플랫폼으로, 학생이 원하는 과목의 검증된 튜터를 쉽고 빠르게 찾아 수업을 예약하고 결제할 수 있는 서비스를 제공한다.

### 1.3 프로젝트 범위
- 회원 관리 (학생/튜터/관리자)
- 튜터 프로필 및 수업 관리
- 수업 예약 및 결제 시스템
- 리뷰 및 평점 시스템
- 관리자 대시보드

### 1.4 기술 스택
| 구분 | 기술 |
|------|------|
| Backend | Spring Boot 3.x, Java 17 |
| Frontend | Thymeleaf, Bootstrap 5, JavaScript |
| Database | MySQL 8.x, MyBatis |
| Security | Spring Security, JWT |
| Build | Gradle |

## 4. 제약사항

### 4.1 기술적 제약사항
- Java 17 이상 필수
- MySQL 8.x 사용
- Spring Boot 3.x 프레임워크 사용

### 4.2 운영 제약사항
- 서비스 운영 시간: 24시간 365일
- 정기 점검: 매월 첫째 주 화요일 02:00~04:00

## 5. 용어 정의

| 용어 | 설명 |
|------|------|
| 튜터 (Tutor) | 수업을 제공하는 강사 |
| 학생 (Student) | 수업을 받는 사용자 |
| 예약 (Booking) | 학생이 튜터에게 수업을 신청하는 것 |
| 수업 (Lesson) | 튜터가 제공하는 과외 서비스 |
| 가용시간 (Availability) | 튜터가 수업 가능한 시간대 |
| 리뷰 (Review) | 수업 완료 후 학생이 작성하는 평가 |
