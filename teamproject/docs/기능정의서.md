## 1. 시스템 개요

### 1.1 시스템 구성도

```
┌─────────────────────────────────────────────────────────────────┐
│                        클라이언트 (Browser)                       │

└─────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────┐
│                      Spring Boot Application                     │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────────┐  │
│  │  Controller │  │  REST API   │  │    Spring Security      │  │
│  │  (Thymeleaf)│  │  Controller │  │    (JWT Authentication) │  │
│  └─────────────┘  └─────────────┘  └─────────────────────────┘  │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │                      Service Layer                          ││
│  └─────────────────────────────────────────────────────────────┘│
│  ┌─────────────────────────────────────────────────────────────┐│
│  │                      MyBatis Mapper                         ││
│  └─────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────┐
│                          MySQL Database                          │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 사용자 유형

| 사용자 유형 | 권한 | 설명 |
|-------------|------|------|
| 비회원 (Guest) | ROLE_ANONYMOUS | 튜터 검색/조회만 가능 |
| 회원 (Member) | ROLE_USER | 수업 예약, 결제, 리뷰 작성 |
| 튜터 (Tutor) | ROLE_TUTOR | 수업 제공, 예약 관리, 스케줄 관리 |
| 관리자 (Admin) | ROLE_ADMIN | 전체 시스템 관리 |

## 3. 상세 기능 정의

### 3.1 인증 (Authentication)

#### FN-AUTH-001: 회원가입
| 항목 | 내용 |
|------|------|
| 기능 ID | FN-AUTH-001 |
| 기능명 | 회원가입 |
| 화면 경로 | /auth/join |
| API 경로 | POST /api/auth/join |
| 설명 | 새로운 사용자를 등록한다 |

**요청 파라미터**
| 파라미터 | 타입 | 필수 | 설명 |
|----------|------|------|------|
| username | String | Y | 이메일 (로그인 ID) |
| password | String | Y | 비밀번호 |
| name | String | Y | 이름 |
| nickname | String | Y | 닉네임 |

**응답**
```json
{
  "success": true,
  "message": "회원가입이 완료되었습니다."
}
```

**비즈니스 로직**
1. 이메일 중복 확인
2. 닉네임 중복 확인
3. 비밀번호 BCrypt 암호화
4. users 테이블에 사용자 정보 저장
5. user_auth 테이블에 ROLE_USER 권한 부여

---

#### FN-AUTH-002: 로그인
| 항목 | 내용 |
|------|------|
| 기능 ID | FN-AUTH-002 |
| 기능명 | 로그인 |
| 화면 경로 | /auth/login |
| API 경로 | POST /api/auth/login |
| 설명 | 사용자 인증 후 JWT 토큰을 발급한다 |

**요청 파라미터**
| 파라미터 | 타입 | 필수 | 설명 |
|----------|------|------|------|
| username | String | Y | 이메일 |
| password | String | Y | 비밀번호 |

**응답**
```json
{
  "success": true,
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIs...",
    "name": "홍길동",
    "nickname": "길동이",
    "roles": ["ROLE_USER"]
  }
}
```

**비즈니스 로직**
1. 이메일/비밀번호 검증
2. Access Token 발급 (유효기간: 1시간)
3. Refresh Token 발급 (유효기간: 7일)
4. Refresh Token을 DB에 저장

### 3.4 예약 관리

#### FN-BOOKING-001: 예약 요청
| 항목 | 내용 |
|------|------|
| 기능 ID | FN-BOOKING-001 |
| 기능명 | 예약 요청 |
| 화면 경로 | /tutors/{id} |
| API 경로 | POST /api/bookings/tutor/{tutorId} |
| 설명 | 학생이 튜터에게 수업을 예약 요청한다 |

**요청 파라미터**
| 파라미터 | 타입 | 필수 | 설명 |
|----------|------|------|------|
| date | String | Y | 수업 날짜 (YYYY-MM-DD) |
| time | String | Y | 수업 시간 (HH:mm) |
| subject | String | Y | 과목명 |
| message | String | N | 튜터에게 전달할 메시지 |

**비즈니스 로직**
1. 해당 시간대 가용시간(availability) 확인
2. 가용시간 상태를 BOOKED로 변경
3. booking 테이블에 예약 정보 저장 (상태: PENDING)
4. 튜터에게 알림 발송

**예약 상태 흐름**
```
PENDING (대기중) → CONFIRMED (승인됨) → PAID (결제완료) → COMPLETED (수업완료)
                ↘ CANCELLED (취소됨)
```
